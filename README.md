# ZalupaReport

ZalupaReport — клиентский мод под Fabric для быстной и удобной обработки репортов на больших серверах (в первую очередь HolyWorld / Lite-Anarchy и подобные).  
Мод НЕ даёт читов и НЕ меняет геймплей, он автоматизирует рутинную работу модератора.

## Основные возможности

### 1. Удобный список подозрительных игроков

При открытии меню `/reportlist` мод:

- Скринит все репорты из GUI.
- Фильтрует по заданным ключевым словам (`detects` в конфиге), например:
  - `CL-BP`
  - `I-B`
  - `MT-PC`
  - `AimANLS`
- Учитывает плейтайм из описания репорта:
  - Общее время в игре
  - Общее активное время
  - Нижние пороги и режимы проверки задаются в конфиге.

Результат отображается в отдельном красивом окошке-слайдере справа, где:

- Каждый найденный игрок — отдельная строка.
- Можно кликнуть по нику — мод кликнет по нужному слоту в `/reportlist` и скопирует ник в буфер обмена.
- Есть кнопка с корзиной — очистка списка.
- Список можно перетаскивать мышкой по экрану.
- Размер и позиция сохраняются в конфиг.

### 2. АвтоВызов (однократный запуск)

Режим для одного репорта:

1. Открываешь `/reportlist`.
2. Нажимаешь **Right Shift**.
3. Мод:
   - Находит первый подходящий репорт по твоим фильтрам.
   - Берёт его (клик по слоту, копирует ник).
   - Пишет `/hm spy <ник>`.
   - Пишет `/find <ник>`.
   - Разбирает ответ `/find` и переходит на нужный сервер:
     - `lanarchyX` → `/ln X` (если `X` отсутствует → `1`)
     - `l2anarchyX` → `/ln120 X`
     - `anarchyX` → `/cn X`
4. После перехода АвтоВызов заканчивается.

### 3. АвтоПроверка (цикличный режим)

Режим полного цикла до победы или пропуска:

1. Включается через секретное меню (см. ниже) → **АвтоПроверка (Цикл)**.
2. Запуск так же из `/reportlist` по **Right Shift**.
3. Поведение:
   - Берёт подходящий репорт.
   - `/hm spy <ник>`.
   - `/find <ник>`, переход на нужный сервер.
   - Ждёт 10 секунд после перехода.
   - Начинает цикл `/playtime <ник>`:
     - Запросы каждые 3 секунды.
     - Анализирует строку `Последняя активность: Xч., Yм., Zс. назад`.
   - Если **последняя активность < 7 секунд**:
     - Игрок активен → `/hm spyfrz` и мод входит в режим ожидания модерации.
   - Если **игрок АФК**:
     - 30 секунд пытается дождаться активности (повторные `/playtime`).
     - Если за 30 секунd активность не снизилась до `< 7с` → считает игрока АФК и завершает репорт по сценарию "АФК".

### 4. Автоматическое завершение репорта

В зависимости от ситуации мод:

- **Игрок АФК (по плейтайму)**  
  → кликает слот **14** в итоговом меню (индекс 13)  
  → отправляет в чат **`afk`** (как обычный игрок)  
  → затем `/hm spy` и возвращается в `/reportlist` за следующим репортом.

- **Сервер пишет `§cИгрок афк проследите сами`**  
  → также считает игрока АФК  
  → слот **14** → сообщение **`afk`** → `/hm spy` → следующий репорт.

- **Бан (`/hm sban...` / `/banip...`) — исходящие или входящие**  
  → кликает слот **16** (индекс 15)  
  → отправляет в чат **`-`**  
  → `/hm spy` → следующий репорт.

- **Анфриз или выход из spy (`/hm unfrz` / `/hm unfreezing` / `/hm spy`)**  
  → кликает слот **14**  
  → отправляет в чат **`-`**  
  → `/hm spy` → следующий репорт.

Во всех вариантах порядок действий при закрытии:

1. `/reportlist`
2. ЛКМ по слоту 47 (индекс 46) — открыть меню действий по репорту.
3. Задержка.
4. ЛКМ по нужному слоту (12/14/16 в зависимости от причины).
5. Задержка.
6. Отправка **`afk`** или **`-`** в чат обычным сообщением (через `sendChatMessage`).
7. `/hm spy` (выход из spy / сброс состояния).
8. `/reportlist` — новый поиск.

### 5. Отслеживание действий модерации

Мод следит за чатами:

- Ловит строки вида:
  - `§bОткрываем репорт ...` — фиксирует начало живой проверки.
  - `§cИгрок афк проследите сами` — принудительное завершение как АФК.
- Отслеживает команды:
  - `hm sban` / `banip`
  - `hm unfrz` / `hm unfreezing`
  - `hm spy`
- Реагирует как на сообщения сервера, так и на команды, которые вводишь **ты сам** (перехватывает исходящие команды через mixin к `ClientPlayNetworkHandler`).

Это позволяет модулю корректно закончить репорт в любом сценарии:
- когда банишь ты,
- когда банит другой модератор,
- когда игрока анфризят,
- когда сервер говорит, что игрок афк.

### 6. Скрытие мусорных сообщений PlayTimeAPI

Во время автоматической проверки плейтайма:

- Все сообщения вида:
  - `------------------PlayTimeAPI------------------`
  - `Активность ...`
  - `Общее время в игре: ...`
  - `Время бездействия: ...`
  - `Последняя активность: ...`
- **НЕ показываются** в чате, чтобы не засорять логи.

Скрытие работает только во время активного сканирования, обычный чат не трогается.

### 7. Управление и горячие клавиши

- Открытие UI списка читеров — автоматически при входе в `/reportlist` (если название окна совпадает с `screenName` в конфиге).
- Запуск сканирования (AutoCall/AutoCheck) — **Right Shift**, когда открыт `/reportlist`.
- Секретное меню — **Right Control**:
  - АвтоВызов (1 раз)
  - АвтоПроверка (цикл)

### 8. Конфиг (Cloth Config + ModMenu)

Через ModMenu → ZalupaReport:

- Заголовок окна `/reportlist` (`screenName`).
- Настройки плейтайма:
  - Общее время в игре, мин.
  - Общее активное время, мин.
  - Включать/выключать соответствующие проверки.
- Список `detects` — ключевые слова из текста жалобы.
- Высота списка HUD.
- Авто-вызов `/spy` после взятия репорта.
- Автопечать `/playtime` после клика по репорту (для ручной проверки).
- Очистка списка при закрытии отчётного экрана.

## Технические детали

- **Версия Minecraft**: 1.20.1  
- **Loader**: Fabric Loader 0.14+  
- **API**: Fabric API  
- **Конфигурация**: Cloth Config  
- **GUI-конфиг**: через ModMenu  
- **Язык**: Java 17  
- **Сборка**: Gradle 8.8 (Wrapper включён, CI на GitHub Actions)

### Важно

Мод рассчитан на работу в паре с HolyModeration и PlayTimeAPI на стороне сервера.  
Команды вроде `/hm spy`, `/hm spyfrz`, `/hm sban`, `/hm unfrz`, `/ln`, `/cn`, `/reportlist`, `/playtime` должны существовать и иметь ожидаемое поведение.

---

## Установка

1. Установите Fabric Loader и Fabric API под Minecraft 1.20.1.
2. Скачайте последнюю сборку `ZalupaReport-*.jar` из GitHub Actions → Artifacts.
3. Положите `.jar` в папку `mods`.
4. Запустите игру.

---

## Отказ от ответственности

- Мод не предоставляет никаких читерских возможностей.
- Он только автоматизирует последовательность действий, которые модератор и так выполняет вручную.
- Используйте на свой страх и риск и только там, где это разрешено правилами проекта.

---

## Кратко

ZalupaReport = мод для модераторов, который:

- Сканирует и фильтрует репорты.
- Автоматически находит репорт, прыгает на нужный сервер, проверяет плейтайм.
- Автоматически жмёт нужные слоты в менюшках `/reportlist`.
- Пишет `afk` или `-` в чат, как настоящий игрок.
- Следит за `/hm sban`, `/banip`, `/hm unfrz`, `/hm spy` и корректно завершает репорт.
- Делает всё это пока ты пьёшь чай.
